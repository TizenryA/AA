# HEARTBEAT.md

每 30 分钟一次：检查是否有待处理任务/结果。

## 检查项目

1. **Cron 任务**：是否有新增/失败/需要处理的运行结果
2. **子代理状态**：是否有完成但未汇报的任务
3. **会话状态**：是否有异常/需要回复的消息
4. **备份链路**：与 GitHub 备份相关的阻塞项是否解决（仓库/权限/冲突等）
5. **上下文占用**：若上下文 >60%，启用 `memory/working-buffer.md` 记录本轮“用户要点 + 回复摘要”

## 汇报格式

- 如果有任务：
  - 汇报任务名称和进度
  - 说明下一步与预计完成时间
- 如果没有任务：
  - 回复 HEARTBEAT_OK

## 防刷屏规则（等待用户决策时）

- 若某项任务**仅差用户一句确认/授权**，且在最近 2 次心跳中已提醒过：
  - 后续心跳默认 **不再重复提醒**，直接回 `HEARTBEAT_OK`
  - 除非出现**新的失败结果/新的告警升级/用户主动追问**

## 验证优先（VBR）

- 对外说“已完成/已修复”前，先做结果验证（看运行结果与实际状态，不只看文案或配置）。
